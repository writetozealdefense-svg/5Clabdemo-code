GCC Financial AI Governance Policies - Knowledge Base
=====================================================

SECTION 1: SAMA Cyber Security Framework (SAMA-CSF)
----------------------------------------------------
The Saudi Arabian Monetary Authority requires all financial institutions
to implement comprehensive cyber security controls. Key requirements include:

- SAMA-CSF 3.1.2: All applications must implement secure coding standards
  including input validation, output encoding, and parameterized queries.
- SAMA-CSF 3.2.1: IAM policies must follow least privilege. No service
  account should have administrative access to production systems.
- SAMA-CSF 3.3.4: All data at rest must be encrypted using approved
  cryptographic standards (AES-256 minimum for sensitive data).
- SAMA-CSF 3.3.6: Software supply chain must be verified. Only signed
  and scanned container images may be deployed to production.

SECTION 2: NCA Essential Cybersecurity Controls (NCA-ECC)
---------------------------------------------------------
The National Cybersecurity Authority mandates the following controls:

- NCA-ECC 1-1-3: Privileged access must be managed through dedicated
  PAM solutions. Service accounts must have time-limited credentials.
- NCA-ECC 2-2-1: Network segmentation must enforce default-deny policies
  between different security domains and workload classifications.
- NCA-ECC 2-3-1: Container workloads must run as non-root with minimal
  Linux capabilities. Privileged containers are prohibited in production.
- NCA-ECC 2-6-1: All security events must be logged to a centralized
  SIEM with a minimum retention period of 12 months.

SECTION 3: Personal Data Protection Law (PDPL)
-----------------------------------------------
Saudi Arabia's PDPL requires organizations to:

- Article 9: Obtain explicit consent before processing sensitive personal
  data including national IDs, financial records, and health information.
- Article 12: Minimize data collection and retention to what is strictly
  necessary for the stated purpose.
- Article 14: Implement logical or physical separation of data belonging
  to different data subjects to prevent cross-contamination.
- Article 19: Deploy technical and organizational measures to prevent
  unauthorized disclosure of personal data.

SECTION 4: Risk Scoring Policy
------------------------------
Customer risk scores are calculated based on:
- Transaction volume anomalies (weight: 30%)
- Geographic risk factors (weight: 25%)
- Counterparty risk assessment (weight: 20%)
- Account age and history (weight: 15%)
- Document verification status (weight: 10%)

Risk levels: Low (0-30), Medium (31-60), High (61-80), Critical (81-100)

Customers with risk scores above 80 require enhanced due diligence (EDD)
and quarterly review by the compliance team.

SECTION 5: AI Governance Requirements
--------------------------------------
All AI systems processing financial or personal data must:
1. Maintain complete audit logs of all model interactions
2. Implement output filtering to prevent PII disclosure
3. Separate system instructions from user inputs
4. Validate and sanitize all inputs before model inference
5. Apply data minimization to context windows
6. Use anonymization/pseudonymization for training data
